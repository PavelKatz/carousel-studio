# carousel-setup — стартовый визард

Интерактивная настройка carousel-studio в текущем проекте. Веди пользователя **по одному шагу за раз**, с короткими понятными вопросами (используй AskUserQuestion, где уместно). Не вываливай всё сразу.

Всё создаётся **в проекте пользователя** (cwd), не в плагине. Дефолты: `brandDir = carousel/brand`, `outputDir = carousel/output`.

## Шаг 0 — Вступление и проверка
Скажи, что за 5 шагов соберём генератор каруселей на его бренде. Проверь, есть ли `carousel.config.json`:
- Нет → новая настройка (шаг 1).
- Есть → спроси: добавить новый референс / поменять голос / переделать заново. Прыгай в нужный шаг.

## Шаг 1 — Бренд и папки
1. Спроси **имя бренда** (короткий slug, напр. `acme` или `my-blog`) → это `owner`.
2. Подтверди дефолтные папки (`carousel/brand`, `carousel/output`) — или дай поменять.
3. Создай:
   - `carousel.config.json`:
     ```json
     { "owner": "<slug>", "brandDir": "carousel/brand", "outputDir": "carousel/output", "genProvider": null }
     ```
   - `<brandDir>/_schema/tokens.schema.json` — скопируй из `${CLAUDE_PLUGIN_ROOT}/examples/brand/_schema/tokens.schema.json`.
   - `<brandDir>/<owner>/references/_index.json`:
     ```json
     { "version": 1, "owner": "<slug>", "default": null,
       "categories": ["course-launch","thought-leadership","case-study","event-promo","personal-story"],
       "references": [] }
     ```

## Шаг 2 — Референсы (ключевой шаг)
**Сначала объясни, что такое референс и какой подойдёт** — кратко, своими словами, по [docs/references-guide.md](../../docs/references-guide.md) (`${CLAUDE_PLUGIN_ROOT}/docs/references-guide.md`). Суть для пользователя:
> Референс — это картинка-образец стиля (твоя прошлая карусель, пост, кадр из бренд-гайда, понравившийся чужой слайд). Из него я вытащу шрифты, цвета, сетку, лого. Нужен **чистый скрин одного слайда**, где видно: заголовок, текст, фон, акцентный цвет. 1 достаточно для старта; 2–3 разных стиля — лучше (обложка / текстовый слайд / финал).

Затем:
1. Попроси дать **1+ картинку-референс** (путь к файлу или папке).
2. Для каждой → вызови skill **`extract-brand-tokens`** (он разберёт её в `tokens.json` + `style.md` и зарегистрирует в `_index.json`). Подтверди `id` и `category`.
3. Поставь `default` в `_index.json` на первый/основной референс.
4. **Минимум 1 референс обязателен.** Если у пользователя нет образца — предложи скопировать пример `${CLAUDE_PLUGIN_ROOT}/examples/brand/acme/references/clean-editorial/` как стартовую точку и поправить цвета/шрифты под него.

## Шаг 3 — Голос бренда
Короткое интервью (3–5 вопросов) → запиши `<brandDir>/<owner>/voice.md`:
- Кто аудитория? О чём контент?
- Тон (экспертный / дружеский / дерзкий / спокойный)?
- Что продаём/к чему ведём (продукт, услуга, экспертиза)?
- Язык контента (ru/en/…)?
- Стоп-слова и табу (что никогда не писать)?
- Ставит ли пользователь подпись-футер сам? (да — агент не клеит её в caption).

Если у пользователя уже есть гайд по голосу — пусть вставит, заверни в `voice.md`. Этот файл агент читает при каждой карусели.

## Шаг 4 — Генеративные фоны (опционально)
Спроси: нужны ли AI-генерируемые фоны/иллюстрации, или хватит чистых фонов из токенов (solid/gradient)?
- **Хватит токенов (рекомендую для старта)** → `genProvider: null`. Готово, ничего не настраиваем.
- **Нужна генеративка** → `genProvider: "replicate"` в конфиг; скажи положить `REPLICATE_API_TOKEN` в окружение (`.env` / shell). Без ключа агент всё равно отрендерит на токенах — не блокер.

## Шаг 5 — Пробный рендер (проверка проводки)
1. Первый раз — подготовь движок: `cd ${CLAUDE_PLUGIN_ROOT}/scripts/carousel-render && npm run setup` (ставит зависимости, Chromium, шрифты). Предупреди, что это разовая загрузка (~100 МБ).
2. Прогон на дефолтном референсе пользователя + примере слайдов:
   ```
   node ${CLAUDE_PLUGIN_ROOT}/scripts/carousel-render/src/cli.mjs \
     --slides ${CLAUDE_PLUGIN_ROOT}/examples/sample-slides.json \
     --tokens <brandDir>/<owner>/references/<default-id>/tokens.json \
     --out carousel/_setup-check --scale 1
   ```
3. Покажи PNG. Если шрифты/цвета попали — настройка завершена.

## Финал
Скажи, как пользоваться:
- **`/carousel <тема | вставленный текст | ссылка>`** — собрать карусель, или просто «сделай карусель про X».
- Новый стиль позже — `/carousel-setup` снова (шаг 2) или «добавь референс».
- Готовые карусели лягут в `<outputDir>/<дата>_carousel-<slug>/` (слайды + caption + alt). Постинг — вручную.

## Принципы
- Один шаг — один вопрос. Не теряй пользователя в простыне.
- Всегда показывай, что создал (путь к файлу).
- Минимум, чтобы поехало: `carousel.config.json` + 1 референс с `tokens.json` + `voice.md`.
