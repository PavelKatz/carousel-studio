# Методология: Instagram-карусели

Source of truth для агента `carousel-creator`. Нарратив, форматы, контракты данных, правила копи/подписи, гейты.

Движок рендера — `${CLAUDE_PLUGIN_ROOT}/scripts/carousel-render/` (см. его README). Библиотека стилей и голос — в проекте пользователя (`carousel.config.json` → `brandDir`, `owner`).

---

## 1. Вход / выход

**Вход — что угодно:** готовый текст/пост, идея, тезисы, ссылка, бриф. Агент сам классифицирует (фаза 0).

**Выход — папка ассетов** `<outputDir>/<YYYY-MM-DD>_carousel-<slug>/`:
```
brief.md              # тип входа, угол, арка, sign-off'ы гейтов, заметки
slides.json           # спека, которую съел рендер
tokens.snapshot.json  # копия токенов использованного референса (провенанс)
caption.md            # подпись для IG (голос бренда) + ## Хэштеги
alt.md                # alt-текст по строке на слайд
slides/NN-*.png       # PNG 1080×1350 по порядку
slides/_bg/NN-bg.png  # только если был генеративный фон
```

Публикация — **ручная**. Агент не постит.

---

## 2. Формат Instagram

- Канвас **1080×1350 (4:5)** — дефолт. Экспорт `--scale 2` → 2160×2700 (резче, IG всё равно сожмёт).
- **6–10 слайдов.** Меньше 5 — лучше пост. Больше 10 — IG не пустит в карусель.
- **Слайд 1 = хук** (обложка), **последний = CTA/вопрос.** Все слайды кропаются под первый — все 4:5.
- Текст — только внутри safe-area (в токенах).

---

## 3. Нарратив-арка

Карусель — мини-сторителлинг: обещание на обложке, развязка в конце.

```
Хук → Контекст/проблема → Слом (контринтуитивное) → Инсайт/каркас
   → Доказательство/пример → Вывод → CTA-вопрос
```

Не обязательно все звенья, но **обещание слайда 1 закрывается к концу**. Одна мысль на слайд. Каждый слайд тянет на следующий. Цель: полезно + мягко продаёт экспертизу/продукт бренда (по `voice.md`), без агрессивных CTA.

---

## 4. Шаблоны слайдов и длина копи

Пять типов (`template`). Лимиты — для дефолтных размеров шрифтов; превышение → текст переполнит слайд. Длиннее — дроби на слайды, не мельчи шрифт.

| template | слоты | назначение | лимиты копи |
|---|---|---|---|
| `cover` | `eyebrow?`, `title`, `subtitle?`, `watermark?` | обложка-хук | title ≤ 40 симв (2–3 строки); subtitle ≤ 90; eyebrow ≤ 24 |
| `body` | `kicker?`, `title?`, `body[]` | абзацы-нарратив | body ≤ 240 симв всего (1–2 абзаца); title ≤ 48 |
| `quote` | `quote`, `attribution?` | панч/цитата | quote ≤ 120 симв |
| `list` | `kicker?`, `title?`, `items[]`, `ordered?` | шаги/пункты/метрики | 4–6 пунктов, каждый ≤ 42 симв (1 строка) |
| `cta` | `kicker?`, `title`, `question?`, `handle?` | финал-вопрос | title ≤ 22; question ≤ 90 |

**Плашки (`highlights`)** — 1–2 ключевых слова на слайд, не больше.

**Номера страниц** — авто на всех слайдах, кроме обложки.

## 5. Контракт `slides.json`

```jsonc
{
  "meta": { "slug": "...", "lang": "ru" },
  "slides": [
    {
      "template": "cover",                    // cover|body|quote|list|cta
      "slots": { "eyebrow": "§ I · …", "title": "...", "subtitle": "...", "watermark": "…" },
      "highlights": ["ключевое слово"],        // 0–2 слова → в accent-плашку
      "background": { "mode": "solid", "color": "bg" }
    }
  ]
}
```

**Рендер и ассеты:** рендери с **оригинальным `tokens.json` референса** (`--tokens <brandDir>/<owner>/references/<id>/tokens.json`). Пути к ассетам в токенах (лого, мотивы) относительны папке референса — из скопированного снапшота не разрешатся, лого молча пропадёт. `tokens.snapshot.json` в джобе — провенанс, не вход рендера.

`background.mode`:
- `solid` — `color` = имя токена (`bg`/`ink`/…) или hex. **Дефолт.** Едет без генеративки.
- `gradient` — `gradient: { angle, stops:[…] }`.
- `image` — `image: "slides/_bg/NN-bg.png"` (путь относительно slides.json) + опц. `overlay`.

Полный контракт слотов — в README движка.

## 6. Фоны: когда какой

- **solid / gradient (дефолт)** — чисто, на бренде, бесплатно, детерминированно. ~80% слайдов.
- **generative image** — только когда слайду реально нужна картинка (обложка-вижуал, метафора). Не на текст-плотных слайдах; если ставишь — добавь `overlay`-скрим.

### Генеративный слой (опционально)
- Включается через `carousel.config.json` (`genProvider`) + env-ключ. Нет ключа → не блокирует: ставь `solid`/`gradient`, отметь в `brief.md`.
- Провайдеры: Replicate (`black-forest-labs/flux-kontext` для style-match по референс-картинке, `recraft-ai/recraft-v3` для иллюстраций), либо fal.ai / Gemini. Контракт «промпт (+опц. референс) → PNG».
- **Промпт фона** = `imagery.promptStyle` из токенов + `slide.imagery_note` + палитра. Всегда добавляй: **«no text, no logo, оставь негативное пространство под текст», 4:5.**
- Сохраняй в `slides/_bg/NN-bg.png`, пропиши в `slides.json`. Рендер кладёт идеальный текст поверх.

## 7. Подпись (`caption.md`) и alt

**Подпись** — в голосе бренда (`voice.md`), на языке контента.
- **Первая строка — хук:** IG обрезает ~после 125 симв.
- Дальше — ценность, в конце — мягкий вопрос/призыв к обсуждению.
- **❌ Никакого футера-подписки** — пользователь ставит сам при публикации.
- **`## Хэштеги`** — отдельной секцией: 5–12, микс брендовых и нишевых по теме.

**alt.md** — по строке на слайд (a11y + индексация).

---

## 8. Quality Gates

- **Гейт 0:** тип входа + ядро + диапазон слайдов.
- **Гейт 1:** арка + кол-во — **до** копи.
- **Гейт 2:** референс.
- **Гейт 3:** копи слайдов — **до рендера** (правки текста здесь дёшевы).
- **Гейт 4:** ассеты (PNG + caption + alt).

Не пропускай 1 и 3.

---

## 9. Визуальные «нельзя»

- ❌ Текст за safe-area.
- ❌ >2 плашки на слайд / плашка на пол-абзаца.
- ❌ Выдуманные факты/метрики. Чувствительные цифры — только относительные, без раскрытия.
- ❌ Фейк-срочность, каунтдауны, «осталось N мест» как хук.
- ❌ Цвета/шрифты не из выбранного `tokens.json`. Нужен другой вид — заведи/правь референс.
- ❌ Эмодзи в тексте слайдов.
- ❌ Авто-публикация.
